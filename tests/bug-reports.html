Bug report id-1:

Active state is removed from button after inserting invalid value to custom input field

Steps to reproduction:

  1. Go to Splitter main page
  2. Click any tip button (e.g. 10%)
  3. Focus on custom tip input and insert letter: a

Expected result:

After step 2, clicked button is active (other background). Inserted value to custom input is invalid so clicked button is still active. Custom input field is empty and not active.

Actual result:

Custom tip field is empty and not active (correct), but active state from selected button has been removed.

Bug report id-2:

Alphanumeric keyboard in mobile version after tapping Number of people or custom tip input

Steps to reproduction:

  1. Open Splitter page in mobile device
  2. Tap Bill field
  3. Tap Number of people field
  4. Tap Custom tip input (tip section)

Expected result:

After tapping Bill field, a keyboard with only digits is showing (correct), but after tapping Number of people or Custom tip field, the opened keyboard is default alphanumeric.

Actual result:

All given fields should have only digits keyboard after tapping them.
